# MD、BioEmu 与 AI 采样

第四章负责把结构预测、采样和分子模拟连接起来，重点关注多组分、多修饰体系的构象探索和后续自由能/稳定性分析。本方法卡把 MD 与 BioEmu/AI 采样拆成可记录的输入、参数、轨迹指标和代表构象选择规则。

## 知识库落点

- 原始课程 PDF、GROMACS 参数、安装脚本、力场说明和可视化/自由能示例保留在 `06_原始学习素材/第四章/`。
- 运行参数、自由能谷底结构抽提、BioEmu 结果解释写入 `04_实验记录/`，优先使用 `模板_MD_BioEmu采样记录.md`。
- 与生成式多肽/抑制剂设计和 MD 验证相关的文献连接到 `chen_design_2024`。
- 与 MD refinement 和机器学习亲和力预测边界相关的文献连接到 `gu_molecular_2023`。

## 可执行流程

1. 明确采样问题：稳定性复核、口袋开合、结合姿态稳定性、突变影响、构象聚类还是自由能面。
2. 选择初始结构：记录实验结构、AlphaFold/Boltz2 预测、对接姿态或上一轮 MD 代表构象。
3. 系统准备：补原子、补缺失、处理修饰、配体参数化、质子化、加水盒、加离子和选择力场。
4. 最小化和平衡：记录约束、温度、压力、步长、时间和是否逐步放开约束。
5. 生产模拟或 AI 采样：记录工具、模型/力场版本、随机种子、输出频率和总采样量。
6. 轨迹/样本分析：计算 RMSD、RMSF、接触、氢键、口袋体积、距离、聚类和代表构象。
7. 结论分级：区分“可作为候选构象”“需要更长采样”“输入参数可疑”“不能用于物理解释”。

## 系统准备检查表

| 检查项 | 必填记录 | 合格标准 |
|:---|:---|:---|
| 初始结构 | 路径、来源、链 ID、模型编号 | 来源可追踪，低置信区已标注 |
| 体系组成 | 蛋白、配体、核酸、金属、糖基化/修饰 | 组分身份和保留规则明确 |
| 质子化 | pH、工具、关键残基状态 | 活性位点和界面电荷假设明确 |
| 配体参数 | 力场、小分子电荷、参数文件路径 | 参数来源可复现，无缺失原子类型 |
| 溶剂和离子 | 水模型、box 类型、离子浓度 | 体系中性或按实验条件设置 |
| 约束 | 位置约束、距离约束、冻结区域 | 约束目的和释放计划明确 |

## MD 运行参数

| 参数 | 说明 |
|:---|:---|
| `force_field` | 蛋白、核酸、配体和水模型版本 |
| `temperature` | 温度和温控方法 |
| `pressure` | 压力和压控方法；NVT 时写明无压控 |
| `time_step` | 时间步长，通常需要说明是否约束氢键 |
| `minimization` | 最大步数、收敛阈值、约束 |
| `equilibration` | NVT/NPT 时长、约束释放策略 |
| `production_time` | 生产模拟总时长和重复数 |
| `trajectory_stride` | 轨迹输出间隔 |
| `random_seed` | 随机种子或“未固定” |

## BioEmu/AI 采样参数

| 参数 | 说明 |
|:---|:---|
| `input_sequence_or_structure` | 输入序列或结构路径 |
| `model_version` | BioEmu 或 AI 采样模型版本 |
| `num_samples` | 样本数或采样轮数 |
| `conditioning` | 是否有配体、突变、约束或构象条件 |
| `seed` | 随机种子 |
| `output_dir` | 结构样本、聚类和统计输出目录 |

## 轨迹指标

| 指标 | 用途 | 解释边界 |
|:---|:---|:---|
| RMSD | 整体稳定性和漂移 | 不能单独证明功能构象 |
| RMSF | 残基层面柔性 | 末端高 RMSF 通常不等于功能变化 |
| H-bond/contact | 界面或口袋相互作用稳定性 | 需定义距离/角度阈值 |
| key distance | 关键残基、配体、金属或催化位点距离 | 阈值要来自结构或文献依据 |
| pocket volume | 口袋开合和可结合性 | 与配体大小和构象聚类一起解释 |
| clustering | 代表构象抽提 | 聚类算法和特征选择会改变结果 |
| free-energy surface | 构象状态比较 | CV 选择决定可解释范围 |

## 代表构象选择

1. 先过滤坏帧：去除明显爆炸、断裂、配体飞出或约束失败的帧。
2. 选择特征：全局 RMSD、口袋残基 RMSD、配体 RMSD、关键距离或接触指纹。
3. 聚类：记录算法、距离度量、簇数或阈值。
4. 代表构象：每个主要簇选 medoid 或离簇中心最近的结构。
5. 回流应用：代表构象可用于 docking、MM-GBSA、Boltz2 对照或结构图。

## AI 采样边界

- BioEmu/AI 采样输出的是构象样本或构象分布近似，不是带真实时间尺度的物理轨迹。
- AI 采样适合快速发现候选构象、口袋状态或柔性区域；不适合直接推断动力学速率。
- 如果要讨论时间、扩散路径、能垒或稳定时长，应使用物理 MD 或自由能方法补证。
- AI 采样和 MD 结果冲突时，优先回到输入结构、条件设置、采样覆盖和实验/结构知识复核。
- MD refinement 不必然提高机器学习亲和力预测，记录时要区分靶点、口袋柔性、初始 docking pose、模拟时长和 ML 模型输入特征。
