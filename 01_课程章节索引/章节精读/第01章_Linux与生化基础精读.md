# 第01章 Linux与生化基础精读

## 本章定位

这一章是后续对接、分子模拟、结构预测和蛋白设计的运行底座。核心不是“学会所有 Linux 命令”，而是建立三件事：可运行的计算环境、能读懂生物分子文件、能定位输入输出和错误。

## 核心概念

- 计算硬件优先关注 NVIDIA GPU、显存、驱动、CUDA 和 Linux 兼容性；没有本地 GPU 时，要明确云服务器或远程机器的替代方案。
- Linux/Ubuntu 是大多数分子建模工具的默认运行环境；Windows 主要承担文件管理、可视化和结果整理。
- 生化文件格式要能区分序列、结构、配体和轨迹：FASTA/A3M 负责序列或 MSA，PDB/mmCIF 负责结构，SDF/MOL2/SMILES 负责小分子，DCD/XTC/TRR 负责轨迹。
- 环境管理优先使用 conda/mamba 或项目独立环境，避免把不同工具的依赖混装到一个全局 Python。

## 可执行流程

1. 确认计算资源：GPU 型号、显存、驱动、CUDA、磁盘空间。
2. 准备 Linux 环境：Ubuntu 22/24 或远程服务器，确认 shell、包管理器和网络。
3. 建立项目目录：原始数据、输入、输出、日志、脚本分开保存。
4. 建立软件环境：为 docking、MD、结构预测分别创建环境，记录安装命令。
5. 用小样例跑通工具：不要直接上大任务，先确认输入格式和输出路径。

## 易错点

- 把 Windows 路径直接复制到 Linux 命令里，导致空格、反斜杠或盘符错误。
- 混淆蛋白结构文件和序列文件；很多 AI 工具接受 FASTA/YAML，而传统对接常需要 PDB/PDBQT/SDF。
- 只记录结果文件，不记录命令、参数和环境版本，后续无法复现。
- 看到 GPU 机器就默认可以跑所有模型，实际还受显存、CUDA、驱动和模型依赖限制。
