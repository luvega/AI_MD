# 第 7 章 VibeCoding、Claude Code 与 AI Agent 工作流

## 本章导读

本章关注工具链和 Agent 工作方式。它不直接产生药物设计结论，但决定课程资料、代码、实验记录、Zotero 文献和在线书籍如何被持续维护。

## 学习目标

- 能说明 LLM Wiki 的 raw/wiki/schema 三层结构。
- 能区分 ingest、takenote、query、lint、zotero 和 update 的职责。
- 能把 AI Agent 输出纳入可验证的知识库维护流程。

## 知识图谱入口

- 章节来源：`01_课程章节索引/章节精读/第07章_VibeCoding与ClaudeCode精读.md`
- 运行手册：`00_项目说明/LLM Wiki运行手册.md`
- 插件说明：`00_项目说明/插件与Skills调用说明.md`
- AI 评测：`07_研究工作台/AI回归评测集.md`

## 核心概念

| 概念 | 本章定位 |
|:---|:---|
| raw/wiki/schema | 原始资料、解释层和操作规则的分离 |
| agent workflow | 用固定流程处理摄入、查询、写入和校验 |
| provenance | 路径、Zotero key、BibTeX key 和维护报告 |
| regression eval | 用标准问题验收 AI 是否真的能用知识库 |

## 方法流程

1. 先读根 `index.md` 和相关 `_index.md`。
2. 根据任务进入章节、方法、文献、实验或研究工作台。
3. 写入知识后更新索引、日志和维护报告。
4. 运行 wiki 校验、图谱体检和在线书籍校验。
5. 对长期问题纳入 AI 回归评测集。

## 关键文献与 BibTeX key

本章以项目工作流和工具链为主，首版不挂接正式 BibTeX key。涉及具体方法证据时回到对应技术章节。

## 实验/练习入口

- 用一个问题从 `index.md` 进入对应章节和方法卡。
- 尝试回答后检查是否包含路径、BibTeX key、边界和下一步。
- 将高价值问题登记到 AI 回归评测任务。

## 使用边界与常见误读

- AI 回答不能替代来源核对。
- 维护工具只检查结构问题，不负责生成新的研究 claim。
- 工具链章节不应和药物设计方法结论混写。
