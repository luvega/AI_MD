# 第 2 章 结构来源、PyMOL 与 Chimera 可视化

## 本章导读

本章把结构文件转化为可检查、可解释的研究对象。结构来源、链选择、缺失区域和可视化复核会影响后续 docking、MD、亲和力预测和蛋白设计。

## 学习目标

- 能区分实验结构、AlphaFold 预测结构和复合物预测结构的用途。
- 能用 PyMOL/Chimera 检查链、配体、口袋、缺失残基和冲突。
- 能说明结构置信度为何不能直接等价于功能结论。

## 核心概念

| 概念 | 本章定位 |
|:---|:---|
| 结构来源 | PDB/mmCIF、AlphaFold2、AlphaFold3 和其他预测模型 |
| 链与生物装配 | 决定复合物、生物学单位和计算体系是否一致 |
| 置信度 | pLDDT、PAE 等指标用于判断结构可用性，不直接证明活性 |
| 可视化复核 | 在进入计算前发现明显结构问题 |

## 方法流程

1. 确认结构来源、版本、链 ID、物种和生物学单位。
2. 检查缺失区域、低置信区域、配体、金属、水分子和辅因子。
3. 用 PyMOL/Chimera 观察口袋、界面和潜在冲突。
4. 把结构处理假设写入后续实验记录，而不是只保存处理后的结构文件。

## 实验/练习入口

- 选择一个蛋白结构，记录 PDB/mmCIF 或预测模型来源。
- 用可视化工具标注链、配体、缺失区域和口袋。
- 写出该结构是否适合进入 docking 或 MD 的判断。

## 使用边界与常见误读

- 预测结构可用于假设生成，但低置信区域不适合作为强结论依据。
- 可视化截图不能替代结构来源、参数和处理步骤记录。
- 复合物预测结构进入亲和力或 PPI 分析前必须复核链和界面质量。
