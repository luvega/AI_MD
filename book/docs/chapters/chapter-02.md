# 第 2 章 结构来源、PyMOL 与 Chimera 可视化

## 本章导读

结构可视化不是为了做出漂亮图片，而是为了确认后续计算使用的结构假设。一个 PDB/mmCIF 文件、一个 AlphaFold 预测模型或一个复合物结构，只有在链、配体、缺失残基、口袋位置和结构来源都能被说明时，才适合作为对接、亲和力预测或蛋白设计的输入。

本章把 PyMOL 与 ChimeraX 当作结构复核工具，而不是展示工具。读者需要通过它们确认结构对象、活性位点、配体位置、模型叠合和截图记录。截图在这里的作用是保存复核证据，不能替代来源说明或实验验证。

第 3 章的 box 定义、第 5 章的复合物输入和第 6 章的设计约束都依赖本章的结构判断。因此，本章完成后应留下结构来源表、口袋截图和人工备注，使后续章节能够直接追溯“为什么选择这个结构”。

可视化复核尤其适合训练“看见结构”和“解释结构”之间的差别。读者在屏幕上看到的链、配体或口袋，只是观察对象；只有当这些对象被来源、截图、坐标和备注共同约束时，才适合进入计算流程。

## 学习目标

本章目标是把结构图从“看起来合理”转化为可记录的结构证据。完成本章后，读者应能够：

- 能区分 PDB/mmCIF 实验结构、AlphaFold 预测结构和经人工处理的工作结构。
- 能在 PyMOL/ChimeraX 中检查链 ID、配体、活性位点、缺失残基和水/金属离子。
- 能说明结构叠合、局部口袋视图和截图在证据链中的边界。
- 能把一张结构图写成可追溯的图注和实验记录字段。

这些目标决定后续 box、口袋、配体定位和结构叠合是否可信。若链 ID、结构来源或配体状态没有记录清楚，第 3 章和第 5 章的输入解释都会失去基础。

## 核心概念

结构来源与可视化复核的核心概念围绕“结构从哪里来、哪些部分可用、哪些区域需要人工判断”展开。它们共同决定后续计算对象是否成立。

| 概念 | 教材化定义 |
|:---|:---|
| 结构来源 | 结构来源决定证据等级；实验结构、预测结构和处理后结构必须分开记录。 |
| 链与配体 | 链 ID、配体名、辅因子和金属离子是后续口袋定义和对接准备的关键字段。 |
| 活性位点 | 活性位点应由共晶配体、功能残基、文献或预测口袋共同约束，而不是只凭视觉中心选择。 |
| 结构叠合 | 结构叠合用于比较构象和模型差异，不能自动说明功能等价。 |
| 截图记录 | 截图应记录视角、对象、颜色、选择命令和来源，避免成为不可复现的装饰图。 |

阅读概念表时，应把每个概念对应到一个具体检查动作：链 ID 是否匹配，配体是否保留，口袋是否有来源，预测结构是否存在低置信区域。

这些检查不是形式步骤。结构来源决定证据等级，链与配体决定计算对象，口袋和叠合决定后续参数。只有当这些信息能被截图和表格共同说明时，结构才适合进入虚拟筛选或模型评估。

例如，AlphaFold 预测结构可以提供构象线索，但低置信区域、缺失配体和复合物假设都会限制后续使用。实验结构也不是自动可用：如果生物学组装、突变、共晶配体或金属离子处理不清楚，后续 box 和亲和力输入同样会受到影响。

在药物设计语境中，“结构可用”不是单一判断。一个结构可能适合展示整体折叠，却不适合定义活性口袋；也可能适合说明配体位置，却不适合进行跨物种功能推断。因此，概念表要和具体任务绑定。

## 方法流程

本章流程从结构载入开始，到复核记录结束。关键不是一次性完成可视化，而是在每一步明确当前观察能支持什么判断。

| 步骤 | 输入 | 动作 | 输出 | QC/边界 |
|:---:|:---|:---|:---|:---|
| 1 | 结构文件 | 确认来源、分辨率/置信度和下载日期。 | 结构来源记录。 | PDB ID 或模型来源可追溯。 |
| 2 | 链和配体 | 检查链 ID、配体、缺失区域和非标准残基。 | 结构 QC 表。 | 关键对象已命名。 |
| 3 | 局部口袋 | 围绕配体或功能残基建立局部视图。 | 口袋截图。 | 选择半径和残基列表明确。 |
| 4 | 叠合比较 | 比较实验结构、预测结构或同源结构。 | 叠合视图。 | RMSD/局部差异不被过度解释。 |
| 5 | 输出记录 | 保存会话、命令、图片和人工判断。 | 可复现结构记录。 | 图注说明来源和边界。 |

执行结构复核时，先用一个结构和一个配体完成小样例，再扩展到批量结构或多个模型。小样例应能暴露链名错配、配体缺失、残基编号不一致和口袋假设不清等问题。

写入方法或实验记录时，建议按“结构来源 -> 链/配体 -> 位点/口袋 -> 截图 -> 边界”的顺序描述。这样的顺序能让后续对接和亲和力预测知道输入结构经过了哪些人工复核。

复核流程还应保留“为什么不用某个结构”的理由。教学中常见错误是只保存最终截图，却不记录淘汰结构的链缺失、口袋不完整或配体状态异常；这些失败理由会直接影响后续结构选择。

对于多结构候选，建议读者建立一个结构选择日志。日志中同时保留采用结构、备选结构和淘汰理由，可以避免后续在 docking 或 Boltz2 阶段重新争论结构来源，也方便在新文献出现后回溯更新。

## 实验/练习入口

本章练习的重点是把结构来源与可视化复核转化成可交接记录。练习完成后，读者应能让另一个人根据记录复现从结构选择到口袋截图的人工复核链，并判断是否具备进入第 3 章受体准备与对接的条件。

建议按以下顺序完成：

1. 选择一个 PDB 或 AlphaFold 结构，写出结构来源、链 ID、配体和置信度/分辨率。
2. 用 PyMOL 或 ChimeraX 导出一张口袋图，并记录选择命令。
3. 比较一个实验结构和一个预测结构，只描述观察到的差异，不直接推断活性变化。

完成练习后，应检查记录中是否包含结构来源表、口袋截图和人工备注、失败原因和人工判断。缺少结构来源表、口袋截图和人工备注时，相关内容仍适合作为课堂尝试，不适合写入正式研究结论。

如果练习借用了文献案例或课程范文，应在结构来源与可视化复核记录中明确它只是方法参照或边界样例。在结构来源与可视化复核中，文献案例可以启发流程设计，但不能替代本项目的本地运行结果。

## 使用边界与常见误读

本章最容易被误读的是结构图、叠合图和预测模型。它们可以支持结构假设，但不能单独说明功能等价或结合可靠。

本章使用边界表时，应把结构图、叠合和预测模型都先视为结构假设，再判断是否需要文献或实验支持。

| 易误读对象 | 稳健表述 | 写作处理 |
|:---|:---|:---|
| 漂亮结构图 | 不能替代结构证据。 | 图注必须写清来源、处理步骤和置信度边界。 |
| 预测结构 | 适合提出结构假设。 | 不能默认等同于实验结构或共晶复合物。 |
| 局部叠合 | 提示局部构象差异。 | 功能解释仍需文献、实验或后续计算支持。 |
| 截图 | 是教学和记录材料。 | 不能作为唯一 provenance。 |

结构图的证据边界应停在“可视化复核支持的结构观察”。当结论涉及功能、结合或活性时，需要额外的计算、文献或实验依据。

稳健写法是说明“该结构用于定义候选口袋”“该叠合提示构象差异较小”，而不是直接写成“该结构足以支持功能一致”或“该口袋必然可结合”。

因此，结构复核的稳健输出是“该结构在当前任务中可作为某类输入假设”，而不是“该结构已经说明功能或结合”。当需要讨论功能时，应引入实验结构解析、突变、活性或文献证据。

截图应被视为复核证据的一部分，而不是最终证据本身。截图能帮助定位链、配体和口袋，但精确判断仍要回到结构文件、残基编号、来源文献和后续计算记录。

## 延伸阅读与下一步

完成本章后，应把结构复核结果交给后续计算流程。优先路径如下：

1. 将明确口袋和配体状态的结构交给第 3 章定义 docking box。
2. 将可信复合物或结构假设交给第 5 章构建亲和力模型输入。
3. 将 motif、hotspot 或界面信息交给第 6 章设计约束。

如果结构来源、链 ID 或口袋依据仍不清楚，应先回到本章补齐记录，而不是继续运行更复杂的软件。

读者进入下一章前，应至少准备一张结构来源表和一张口袋复核截图。表格说明结构从哪里来、如何处理；截图说明链、配体和口袋如何定位。两者配套使用，后续对接章节才能回答 box 的来源问题。若结构存在多个候选，应把备选结构和淘汰理由保留在记录中，方便后续重新评估。

若准备进入虚拟筛选，读者还应把口袋定义写成可复制参数，例如中心坐标、box 大小、来源依据和人工备注。若准备进入蛋白设计，则应把 motif、hotspot 或界面残基单独列出。不同后续任务需要不同结构字段，本章记录应提前服务这些交接。

如果后续发现结构来源需要更新，应优先更新结构来源表和截图说明，再同步修改后续章节的输入解释，避免旧结构假设继续传播。
