# 第 1 章 Linux 与生化计算基础

## 本章导读

本章建立生化计算项目的基础工作环境。它不直接产生药物设计结论，但决定后续 docking、MD、Boltz2、RFdiffusion 和 Chai-1 工作能否被复现。

## 学习目标

- 能解释 Linux 文件路径、命令行、环境变量和批处理在计算药物化学中的作用。
- 能区分原始资料、wiki 笔记、实验记录和运行输出。
- 能为后续章节准备可追踪的输入、输出和日志习惯。

## 核心概念

| 概念 | 本章定位 |
|:---|:---|
| 工作目录 | 所有命令、相对路径和输出文件的参照点 |
| 环境变量 | 控制软件、缓存、依赖和可执行程序的查找方式 |
| 文件格式 | FASTA、PDB/mmCIF、SDF、CSV/TSV、YAML 和日志文件是后续方法的共同接口 |
| 可复现记录 | 每次运行要保留输入、参数、版本、输出和失败原因 |

## 方法流程

1. 建立项目目录，明确 raw、wiki、schema 和 experiment 的分层。
2. 用命令行检查输入文件是否存在、格式是否符合下游工具要求。
3. 运行计算任务时保存命令、版本、随机种子和输出路径。
4. 将有长期价值的运行信息沉淀到实验记录模板，而不是只保留临时聊天记录。

## 实验/练习入口

- 建立一个空白实验目录，包含 `inputs/`、`outputs/`、`logs/` 和 `notes/`。
- 练习把一个 FASTA 或 PDB 文件从来源、检查命令、输出文件到记录说明完整写下来。

## 使用边界与常见误读

- Linux 命令执行成功不等于科学结果可信；它只说明任务完成了某个计算步骤。
- 文件路径和命名不规范会导致后续结果不可追踪。
- 首版讲义不复制原始课程文件；需要核对细节时回到 AI_MD 原知识库。
