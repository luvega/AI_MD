# P25 中文教材正文风格指南

本指南用于 AI_MD 在线书籍正文润色。它综合 `Research-Paper-Writing-Skills` 的段落功能、reverse outline、claim-evidence alignment，以及 `academic-chinese-style` 的中文生物医学写作边界。

## 适用范围

- 适用于 `book/docs/chapters/` 的教学正文、资源页说明和维护报告摘要。
- 不改写 `<!-- refs:start -->...<!-- refs:end -->` 内的自动引用列表，引用区由生成脚本统一维护。
- 不改写代码块、图片链接、DOI/URL、文件路径和 manifest 字段。

## 段落规则

- 每段只承担一个功能，第一句说明本段要解决的问题或判断。
- 教学段落优先采用“问题场景 -> 方法动作 -> 证据边界 -> 下一步”的顺序。
- 长流程用表格表达 `输入 | 动作 | 输出 | QC/边界`，不要把所有步骤塞进一个长段落。
- 章节结尾必须指向下一章、研究工作台或实验记录模板。

## P34 教材叙事段落

P34 采用“中等扩写 + 教材讲解”风格。目标不是把章节改成论文综述，也不是把正文改成操作手册，而是让读者能够连续理解“为什么这样做、怎么做、如何判断、下一步接到哪里”。

- 段落先给问题场景，再解释概念或流程；不要直接从术语表开始。
- 表格前必须说明为什么需要这张表；表格后必须说明如何把表格用于判断。
- 每个核心概念至少回答三个问题：约束什么输入、影响什么输出、需要怎样记录。
- 方法流程按 `输入 -> 动作 -> 输出 -> QC/边界` 解释，正文负责讲清判断逻辑，表格负责承载速查信息。
- 代码案例只能解释输入组织、参数替换、输出读取和失败排查；不能把 dry-run 写成真实预测或实验结果。
- 使用边界采用保守语气：模型输出、评分、截图和文献案例优先写作“提示”“支持”“可作为线索”“仍需验证”。
- 结尾段落必须把本章内容转成阅读队列、实验队列、项目池或下一章入口，避免停留在“学完本章”的空泛表述。

本轮参考 `zLanqing/codex-claude-academic-skills` 的中文科研表达、来源分层和术语保留原则，也参考 `SNL-UCSB/paper-writing-skill` 的 topic sentence、paragraph flow、reverse outline 和“先扩写形成完整论证，再压缩”的写作方法。两者只作为写作规则来源，不复制第三方 skill 内容。

## P35 去模板化规则

P35 在 P34 的基础上增加“章节专属化”约束。在线教材可以保留统一栏目，但正文句子不能让读者感觉每章只是替换了几个名词。每个重点段落至少应出现本章自己的对象、输入、QC、边界或下一步任务。

- 导读段落应回答本章独有的判断问题，例如结构来源、docking score、predicted affinity、RFdiffusion/RFD3 设计链或 Chai-1 项目路线，而不是只重复“为什么做、怎么做、如何判断”。
- 表格前后的解释必须服务本章表格。若同一句话能原封不动放入 4 章以上，应改写为带本章对象的句子。
- 代码案例说明应指出当前代码生成的是哪一类记录，例如环境 manifest、口袋截图、候选 triage 表、轨迹指标、预测表或项目优先级表。
- 使用边界段落必须点名本章最容易误读的对象，并说明它最多支持什么表述。
- 延伸阅读段落应说明本章资产如何进入下一章、实验记录、阅读队列或项目池。
- 可读性审计中的 repeated sentences 只作为机器提示；最终判断仍以人工阅读是否自然、是否保留证据边界为准。

## 证据边界

| 证据强度 | 推荐动词 | 避免 |
|:---|:---|:---|
| 直接实验或严格验证 | 表明、证实、证明 | 无来源地使用 |
| 多来源一致模式 | 提示、支持、说明 | 写成决定性结论 |
| 模型预测或评分 | 可能提示、可作为排序线索 | 写成 Kd、IC50、结合强度或活性 |
| 文献案例/课程范文 | 可借鉴、可作为案例 | 写成本项目结果 |

## 术语与 provenance

- `docking score`、`predicted affinity`、`confidence`、`aggregate score`、`RFdiffusion/RFD3`、`ProteinMPNN`、`Chai-1` 保持术语一致。
- 章节正文不展示内部引用键或本地文献库条目编号；这些信息保留在 `references/` 元数据中，用于生成和 provenance 追踪。
- 原始 PDF 和补充材料只作为来源，不直接复制图表到在线书籍。
