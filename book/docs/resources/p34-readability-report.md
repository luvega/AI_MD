# P34 中文教材可读性增强报告

## 本轮目标

P34 的目标是把 8 个主章节从“导航型讲义骨架”推进到“可连续阅读的教材正文”。本轮不新增科学事实、不新增文献结论、不重做 Imagegen 图，也不改变引用区、代码块、图片路径、DOI/URL、BibTeX key 或 Zotero item key。

采用的写作取向为“中等扩写 + 教材讲解”。这意味着正文要解释为什么这样做、怎么做、如何判断、下一步接到哪里；但不把章节扩写成完整专著，也不把课程讲义改成论文综述。

## 参考规则提炼

本轮参考 `zLanqing/codex-claude-academic-skills` 的中文科研表达、术语保留、来源分层和审稿式边界控制原则。可迁移到 AI_MD 的做法是：中文表达保持克制，英文术语和模型名稳定保留，预测和评分只写成线索，文献案例和本项目结果必须分层。

本轮同时参考 `SNL-UCSB/paper-writing-skill` 的段落组织原则。可迁移到教材正文的做法是：先建立 topic sentence 序列，再扩写为完整段落；每段只承担一个教学任务；句与句之间需要有原因、对比、后果或细化关系；写完后用 reverse outline 检查段落是否服务本节目标。

## 改写策略

- `本章导读`：从问题场景进入，说明本章为什么存在、要解决什么判断问题、后续章节如何复用。
- `核心概念`：表格前增加阅读方式，表格后说明如何把概念转成记录字段和判断路径。
- `方法流程`：表格前解释流程为什么按输入、动作、输出和 QC 组织，表格后说明如何从最小样例进入批量任务。
- `代码案例与软件操作`：保留代码块和截图，新增“教材化阅读提示”，解释参数替换、输出解读和失败排查。
- `使用边界与常见误读`：继续使用保守语气，强调 score、affinity、design、Chai-1 和文献案例不能被写成本项目结果。
- `延伸阅读与下一步`：把章节结尾从列表改成学习路线和研究资产转化说明。
