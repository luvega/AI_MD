# 研究问题与项目池

本页把课程材料转成个人研究工作台候选。每个项目都必须先区分“文献案例”“方法假设”“本项目真实运行结果”，再决定是否进入实验队列。

## 项目池总览

## 项目进入执行的门槛

| 门槛 | 必填内容 |
|:---|:---|
| 研究对象明确 | 靶点、配体/蛋白面板、物种、疾病或药理背景 |
| 数据来源明确 | PDB/mmCIF、FASTA、SMILES/SDF、候选库、已有活性数据 |
| 方法边界明确 | 哪些输出只用于排序，哪些可进入结论，哪些必须实验验证 |
| 模板已选 | 对接、MD、Boltz2、RFdiffusion、ProteinMPNN 或 Chai-1 模板 |
| 文献锚点已查 | Zotero item key 和 BibTeX key 进入记录；未锚定时写“待补” |

## 当前排序建议

1. 先做 `proj:boltz2-affinity-triage` 或 `proj:ape1-natural-product-screen`，因为已有模板最完整，能最快形成真实运行记录。
2. 再做 `proj:chai1-ppi-panel`，但必须先补 Chai-1 正式文献/官方锚点，避免误用 aggregate score。
3. `proj:baba-binder-design` 和 `proj:ido1-reproducible-vs` 更适合作为课题申请或训练项目，进入前需要明确计算资源和真实输入数据。
