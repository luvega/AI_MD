# 概念关系规范

本规范用于避免 wiki 只留下模糊的“相关”，而丢失关系类型。新增或更新页面时，优先使用 typed relation。

## 关系类型

| 关系 | 含义 | 示例 |
|:---|:---|:---|
| `supports` | 文献、实验或资料支持某个判断 | `passaro_boltz-2_2025 supports Boltz2 亲和力解释规则` |
| `contradicts` | 新资料与旧判断存在冲突 | `新 benchmark contradicts 旧 score 解释` |
| `updates` | 新页面或资料更新旧页面 | `P5 报告 updates Zotero 候选表` |
| `depends_on` | 方法或结论依赖前置条件 | `Boltz2 亲和力解释 depends_on 输入 YAML 和结构置信度` |
| `derived_from` | 页面内容来自某个原始资料或维护报告 | `章节精读 derived_from PDF 全文提取` |
| `applies_to` | 方法适用于某个章节、任务或场景 | `ProteinMPNN applies_to RFdiffusion 后序列设计` |
| `extends` | 页面扩展已有方法线 | `LigandMPNN extends ProteinMPNN` |
| `needs_review` | 需要人工复核 | `OCR page needs_review 原 PDF` |

## frontmatter 写法

```yaml
relations:
  - type: supports
    target: "../02_方法笔记/Boltz2亲和力预测.md"
    evidence: "passaro_boltz-2_2025"
  - type: needs_review
    target: "06_原始学习素材/PDF OCR质量收敛报告.md"
    evidence: "OCR 质量=需人工整理"
```

## claims 规则

- `claims` 可以记录 BibTeX key、本地 claim id 或维护报告 id。
- 有明确论文支撑时，优先写 BibTeX key。
- 只有聊天推断、未核验实验结果或 OCR 噪声时，不写成确定 claim，标注 `待确认`。
