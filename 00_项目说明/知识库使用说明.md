# AI_MD 知识库使用说明

## 日常入口

- LLM Wiki 总入口：打开根目录 `../index.md`，先看章节、方法、文献、实验、附件、维护报告和开放问题。
- 操作时间线：打开根目录 `../log.md`，查看最近 ingest、query、update、lint、zotero、ocr、git 和 maintenance 记录。
- LLM Wiki 规则：需要理解第二大脑工作方式时，先读 `LLM Wiki Agent说明.md`、`LLM Wiki运行手册.md` 和 `概念关系规范.md`。
- Obsidian 浏览首页：打开 `Obsidian入口.md`，再进入章节、方法、文献、实验和附件索引。
- 宽泛整理、下一步计划、判断该用哪个插件或 skill：使用 `/ai-md-router`。
- 新来源摄入：使用 `/ingest-source`；它会规划读取来源、调用 `/takenote` 写入、再用 `/update-vault` 验收。
- 基于知识库问答：使用 `/query-wiki`；长期有价值的答案再沉淀为笔记。
- 周期健康检查：使用 `/wiki-lint`；它负责重复概念、孤立页、矛盾候选和待补来源检查。
- 记录想法、摘要 PDF、整理运行结果：使用 `/takenote` 或自然语言“记一下”。
- 按主题查 Zotero 并建立文献笔记：使用 `/zotero-literature-link`。
- 批量重建索引和检查断链：使用 `/update-vault`。
- 查看课件全文提取：先打开 `../06_原始学习素材/PDF全文提取总览.md`，再进入对应章节的 `全文.md`、`pages/`、`ocr/` 和 `extraction-report.md`。
- 研究工作台入口：打开 `../07_研究工作台/_index.md`，再进入实体索引、claims 矩阵、项目池、阅读队列、实验队列、输出视图和 AI 回归评测集。
- 在线书籍入口：打开 `../book/docs/index.md`，查看 MkDocs Material 课程讲义；课程资源位于 `../book/docs/resources/`，包括代码案例、截图索引、Imagegen prompt 和复现实验资源；构建命令为 `python -m mkdocs build -f book/mkdocs.yml --strict`。
- 在线书籍 GitHub Pages：远端仓库为 `https://github.com/luvega/AI_MD.git`，部署后站点为 `https://luvega.github.io/AI_MD/`；workflow 位于 `../.github/workflows/deploy-book.yml`。
- 图谱体检：运行 `python tools/graph_health.py . --json --stale-days 180`，检查实体数、typed relations、孤立页、过期 `last_reviewed`、缺 BibTeX 的 Zotero 映射和文献笔记 key 缺口。
- 在线书籍校验：运行 `python tools/validate_online_book.py --map book/book_map.toml --book-root book/docs --require-nature-refs --require-imagegen`，检查章节区块、来源路径、Nature 引用区、Imagegen 资源、BibTeX key、书内链接和 raw source 链接。
- 在线书籍引用重建：运行 `python tools/update_book_references.py --map book/book_map.toml --book-root book/docs --references references/references.bib --zotero-map references/zotero-map.tsv`，从 BibTeX/Zotero 映射更新章节引用卡片和附录 C。

## 推荐检索顺序

1. 读取根目录 `CLAUDE.md`。
2. 读取根 `../index.md` 和最近 `../log.md`。
3. 如果任务涉及插件或多个模块，读取 `插件与Skills调用说明.md`。
4. 根据任务类型读取对应目录 `_index.md`。
5. 如需查课件原文，读取 `06_原始学习素材/PDF全文提取总览.md`；如需学习路线，读取 `01_课程章节索引/章节精读/_index.md`。
6. 如需做研究决策，先读取 `07_研究工作台/实体索引.md` 和 `07_研究工作台/证据与claims矩阵.md`。
7. 如需生成课程讲义或在线书籍内容，读取 `book/book_map.toml`、`book/docs/index.md`、对应章节页面和 `book/docs/resources/`。
8. 读取具体方法笔记、文献笔记或实验记录。
9. 如需引用文献，检查 `references/zotero-map.tsv` 和 `references/references.bib`。

## 文献键说明

- Zotero item key：本地 Zotero 条目 ID，例如 `FF4V8LYV`。
- BibTeX key：写作引用 ID，例如 `passaro_boltz-2_2025`。
- 如果 Zotero 导出的 BibTeX key 有重复，本项目在 `references.bib` 中使用本地 alias，并在映射表中说明来源。

## Obsidian 使用

- 当前只提供 Markdown 首页和模板目录，不写入 `.obsidian/` 配置。
- 模板位于 `Obsidian模板/`，可复制到对应目标目录后再改 frontmatter。
- 图谱和反链优先依赖统一 frontmatter、`related` 字段和目录 `_index.md`。
- 如果使用 Dataview，可以优先围绕 `wiki_role`、`source_count`、`last_reviewed`、`claims` 和 `relations` 建表。
- 研究图谱优先从 `../07_研究工作台/实体索引.md` 和 `../07_研究工作台/证据与claims矩阵.md` 读取；不要直接把补充 PDF 的文献案例写成本项目结果。
- 在线书籍优先使用 `../book/book_map.toml` 追踪来源；页面中不要直接链接或嵌入 `06_原始学习素材/` 中的原始 PDF、图片或压缩包。Imagegen 图像和软件截图必须登记到 `../book/docs/resources/` 的 manifest。
- 推送 GitHub Pages 前先运行在线书籍校验和 MkDocs strict build；公开发布前仍需版权审查、图表替换和本地路径清理。
- 本项目已经允许本地 Git 版本史；默认记录 Markdown wiki、schema、skills、BibTeX、TSV 和结构化文本，不记录大型不可变原始二进制资料，不配置 remote、不 push。
