# P40 旧版在线图书删除记录

本次维护的目标是清掉 P20-P39 阶段形成的 8 章旧版在线教材发布层，避免它继续影响当前 12 章《AI 辅助药物设计：从分子建模到研究工作台》教材生成。

## 使用材料

- `大纲.md`：确认当前教材结构已调整为 12 章，并明确旧版 `book/docs/` 只是历史发布层。
- `AGENTs.md`：确认当前写作流程需要先经过逐章大纲和正文草稿。
- `CLAUDE.md`：确认原始素材边界、知识库目录规则和维护验收方式。
- `git status --short`、`git ls-files`、`rg` 检索结果：确认旧版在线书工程、部署 workflow、旧书专用脚本和测试的跟踪状态。

## 已删除范围

| 类型 | 路径 | 说明 |
|:---|:---|:---|
| 旧版发布工程 | `book/` | 包含旧版 `book/docs/` 源页面、`book/site/` 本地 HTML 构建、MkDocs 配置、章节映射、旧图像和资源页。 |
| 自动部署 | `.github/workflows/deploy-book.yml` | 旧 GitHub Pages 发布流水线，依赖已删除的 `book/`。 |
| 旧书校验和生成脚本 | `tools/validate_online_book.py`、`tools/polish_book_chapters.py`、`tools/update_book_references.py`、`tools/apply_p24_book_assets.py` | 绑定旧 8 章发布层，不再作为当前教材生成工具。 |
| 旧书审计脚本 | `tools/audit_book_figures.py`、`tools/audit_book_readability.py`、`tools/audit_book_review_readiness.py` | 绑定旧版图注、可读性和 P39 审稿就绪规则。 |
| 旧书测试 | `tests/test_validate_online_book.py`、`tests/test_polish_book_chapters.py`、`tests/test_update_book_references.py`、`tests/test_audit_book_figures.py`、`tests/test_audit_book_readability.py`、`tests/test_audit_book_review_readiness.py` | 旧发布层消失后会误导当前测试入口。 |

## 保留范围

- `chapters/` 保留，作为当前 12 章教材生成工作区。
- `大纲.md` 保留，作为整本教材结构的当前来源。
- `06_原始学习素材/` 保持只读，不移动、不删除、不上传 GitHub。
- `00_项目说明/book-stage-reports/` 和 P20-P39 维护报告保留为历史记录，但不再作为新正文来源。
- `references/`、`01_课程章节索引/`、`02_方法笔记/`、`03_文献笔记/`、`04_实验记录/`、`07_研究工作台/` 保留，继续作为来源和证据层。

## 当前规则

当前正文生成先落在 `chapters/chapter-XX/本章大纲.md` 和 `chapters/chapter-XX/正文.md`。只有 12 章结构、来源材料、引用边界和图表需求确认后，才重建新的 `book/` 发布层。

重建前不要调用旧路径：

```text
book/book_map.toml
book/docs/
book/site/
book/mkdocs.yml
tools/validate_online_book.py
tools/polish_book_chapters.py
```

## 待确认项

- 是否需要同步清理远端 GitHub Pages 上已经发布的旧站点，需要后续通过 GitHub Pages 设置或新 commit/部署策略处理。
- 新 `book/` 发布层的工具链、目录结构、校验脚本和 GitHub Pages workflow 需要在 12 章正文稳定后重新设计。
