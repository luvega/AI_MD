# P26 原始素材目录本地保留

## 更新范围

- `06_原始学习素材/` 作为本地 raw source 目录继续保留。
- GitHub 仓库只保留 `06_原始学习素材/.gitkeep` 占位文件，目录内容不再上传。
- `.gitignore` 已新增整目录忽略规则，避免后续 PDF、OCR、全文提取、torrent、解包文件和其他 raw payload 被误加入 Git。
- 已从 Git 索引移除既有 raw source 文件；该操作只影响版本追踪，不删除本地文件。

## 边界说明

- 本地知识库仍可读取 `06_原始学习素材/` 中的原始课件、全文提取和 OCR 结果。
- 远端 GitHub 仓库不再保证 raw source 路径存在；在线书籍不得直接链接或嵌入 raw source 文件。
- `references/`、wiki 笔记、实验记录、研究工作台、在线书籍和维护脚本仍按普通项目文件管理。

## 后续注意

- 新增原始素材时直接放入本地 `06_原始学习素材/`，不要 `git add` 该目录内容。
- 如需共享原始素材，应使用 Zotero、网盘或受控内部存储，并在 wiki 中记录来源和访问边界。
- 如果将来需要公开教材，应继续只发布 `book/` 生成内容和可公开资源，不发布 raw source 目录。
