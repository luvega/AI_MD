# P30 图示与版面升级报告

## 更新结论

P30 已把在线书籍的图示体系从“Imagegen 位图为主”升级为“Mermaid source of truth + Imagegen 辅助图 + scientific-schematics prompt”的三层结构。8 个主章节都新增了 Mermaid 结构图，并在课程资源中新增 `Mermaid 图示与示意图设计` 页面，记录每章图示和后续生成科学示意图的 prompt。

本轮不重新生成 Imagegen 图，不复制原始 PDF 图表，不新增外部图片。所有新增图示都是 Markdown/Mermaid 文本，便于 Git diff、AI 读取和后续重绘。

## 关键改动

| 模块 | 改动 |
|:---|:---|
| MkDocs | `book/mkdocs.yml` 启用 Mermaid custom fence，并把资源页加入导航。 |
| 章节正文 | 第 1-8 章在 Imagegen 图谱之后新增 `Mermaid 结构图`。 |
| 课程资源 | 新增 `book/docs/resources/mermaid-schematics.md`，保存每章 Mermaid 图和 scientific-schematics prompt。 |
| 校验器 | `tools/validate_online_book.py` 新增 `--require-mermaid`，检查 Mermaid fenced block、`accTitle` 和 `accDescr`。 |
| GitHub Pages | `.github/workflows/deploy-book.yml` 使用完整在线书籍校验参数。 |
| 测试 | `tests/test_validate_online_book.py` 增加 Mermaid accessibility 校验用例。 |
| 版本 | `book/book_map.toml` 标记为 `v0.4-p30-mermaid-schematics`。 |

## 每章图示目标

| 章节 | Mermaid 图示重点 | 后续示意图方向 |
|:---|:---|:---|
| 第 1 章 | Linux 项目记录闭环 | 环境检查到实验记录流程 |
| 第 2 章 | 结构证据复核链 | 结构来源、叠合和截图记录 |
| 第 3 章 | docking evidence funnel | score 只能支持排序，pose QC 必须记录 |
| 第 4 章 | MD interpretation loop | 轨迹指标到代表构象，不越界到机制证明 |
| 第 5 章 | affinity prediction interpretation | predicted affinity、confidence、calibration 联合解释 |
| 第 6 章 | protein design validation chain | 生成候选、序列设计、回折叠、界面 QC 和实验交接 |
| 第 7 章 | agent verification loop | brief、执行、验证、沉淀和复核 |
| 第 8 章 | research workbench roadmap | 研究问题、证据、方法路线、claim 和输出 |

## 后续交接

P31 可以直接复用本轮 Mermaid 图示作为 dry-run 代码案例和实验记录模板的字段来源。第 3/5/6/8 章尤其应把 Mermaid 中的 decision node 转成记录模板字段，例如 `pose_qc_passed`、`calibration_available`、`interface_qc_passed` 和 `evidence_maturity`。
