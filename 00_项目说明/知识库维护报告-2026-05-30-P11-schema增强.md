# P11 schema 增强

## 范围

本轮只做 schema 增强，不扩写研究内容，不移动原始资料。

## 已补齐页面

| 类型 | 页面 |
|:---|:---|
| 根入口 | `index.md`, `log.md` |
| LLM Wiki 规则 | `LLM Wiki Agent说明.md`, `LLM Wiki运行手册.md`, `概念关系规范.md` |
| 方法卡 | `AI多组分对接与虚拟筛选.md`, `MD_BioEmu_AI采样.md`, `Boltz2亲和力预测.md`, `RFdiffusion与蛋白设计.md` |
| 文献笔记 | `Boltz2亲和力预测.md`, `分子对接与虚拟筛选.md`, `亲和力模型与肽结合排序.md` |
| 维护报告 | `P9-LLM-Wiki-Agent落地`, `P10-wiki-lint健康检查`, `P11-schema增强` |

## 增强内容

- 给核心方法卡补 `wiki_role: method`、`source_count`、`last_reviewed`、`claims`、`relations`。
- 给核心文献笔记补 `wiki_role: literature`，并用 `supports` 回连对应方法卡和章节精读。
- 给根索引、日志和三份 LLM Wiki 规则页补 `claims` 与 typed relations。
- 给 P9/P10/P11 维护报告建立 `updates`、`applies_to` 和 `needs_review` 关系。

## 保留问题

- 模板页仍允许 `source_files: []`，不在 P11 中改成实体来源。
- 旧维护报告中保留 `待确认`、`暂缓`、`需人工` 等状态词，作为开放问题，不在 schema 增强阶段改写。
- 其余旧笔记可在 P12 按目录逐步补齐 typed relations。
