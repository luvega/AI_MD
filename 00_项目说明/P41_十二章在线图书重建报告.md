# P41 十二章在线图书重建报告

## 本轮目标

本轮把《AI 辅助药物设计：从分子建模到研究工作台》的 12 章正文整理成新的在线教材发布层。用户明确要求不上传本章大纲，只上线正文内容；配色风格采用蓝白配色。

## 使用材料

- 根目录 `大纲.md`：用于确认 12 章顺序和章节标题。
- `chapters/chapter-01/正文.md` 至 `chapters/chapter-12/正文.md`：作为在线教材正文的唯一内容来源。
- 各章 `assets/`：只复制正文实际引用的发布资源，不复制 Markdown 写作资产。

`chapters/chapter-XX/本章大纲.md` 仅作为本地写作准备资料，本轮不进入 `book/`。`06_原始学习素材/` 继续只读并保留在本地，不复制到发布层，也不在页面中暴露原始路径。

## 主要更新

- 新建 `book/mkdocs.yml`，使用 MkDocs Material 重建 GitHub Pages 发布层。
- 新建 `book/docs/index.md` 和 `book/docs/chapters/chapter-01.md` 至 `chapter-12.md`，覆盖全书 12 章正文。
- 新建 `book/docs/stylesheets/blue-white.css`，采用白底、蓝色导航、浅蓝提示块和移动端自适应排版。
- 新建 `tools/sync_online_book.py`，从 12 章正文同步发布层，并过滤待作者确认项、旧发布路径、原始素材路径和本章大纲。
- 新建 `tools/validate_online_book.py` 与 `tests/test_online_book_publish.py`，检查章节完整性、链接、蓝白主题和禁用内容泄漏。
- 恢复 `.github/workflows/deploy-book.yml`，用于 push 到 `master` 后自动构建并部署 GitHub Pages。

## 验收结果

已通过以下本地验收：

```powershell
python tools\sync_online_book.py
python tools\validate_online_book.py
python -m pytest -q
python -m mkdocs build -f book\mkdocs.yml --strict
python tools\graph_health.py . --json --stale-days 180
```

`mkdocs build --strict` 通过。Material for MkDocs 输出了面向未来 MkDocs 2.0 的上游兼容性提示，不影响本轮构建。

图谱体检仍保留既有提醒：部分旧页面缺少 `last_reviewed`，少数页面处于孤立状态，个别文献页缺 Zotero 或 BibTeX 锚点。这些不是 P41 发布层新增问题。

## 待确认项

- GitHub Pages 需要在远端仓库设置为 GitHub Actions 发布；提交并推送后 workflow 才会正式上线。
- 公开发布前仍建议人工抽查 12 章正文、图示授权、外部链接和术语表述边界。
