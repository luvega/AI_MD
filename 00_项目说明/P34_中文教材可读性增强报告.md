# P34 中文教材可读性增强报告

## 本轮目标

P34 按“中等扩写 + 教材讲解”方案，把 8 个在线书籍主章节从较短的导航型讲义骨架扩写为可连续阅读的教材正文。本轮不新增科学事实、不新增文献结论、不重做 Imagegen 图，也不改变引用区、代码块、图片路径、DOI/URL、BibTeX key 或 Zotero item key。

## 参考规则

本轮参考 `zLanqing/codex-claude-academic-skills` 的中文科研表达、术语保留、来源分层和过强表述控制原则；同时参考 `SNL-UCSB/paper-writing-skill` 的 topic sentence、paragraph flow、reverse outline 和先扩写再压缩的写作方法。外部仓库只作为写作规则来源，AI_MD 不保存第三方 skill 副本。

## 已完成更新

| 更新项 | 说明 |
|:---|:---|

| 风格指南 | 在 `book/docs/resources/style-guide.md` 中新增 P34 教材叙事段落规则。 |

| 审计脚本 | 新增 `tools/audit_book_readability.py`，统计非表格正文长度、重点区块长度、重复句和过强表述。 |
| 单元测试 | 新增 `tests/test_audit_book_readability.py`，覆盖正文统计和阈值失败场景。 |

## 审计结果

P34 前，每章非表格/非代码/非引用正文最小值为 1723，平均值为 1976.5；重点区块最小值为 31，审计错误数为 45。P34 后，每章非表格/非代码/非引用正文最小值为 3630，平均值为 3805.5；重点区块最小值为 364，审计错误数为 0。

验收命令：

```powershell
python tools\audit_book_readability.py --book-root book\docs --min-prose-chars-per-chapter 3500 --min-priority-section-chars 350
```

## 边界说明

第 3/5/6/8 章继续保护高风险表述：`docking score`、`predicted affinity`、`RFdiffusion/RFD3`、`Chai-1 aggregate score` 和文献案例均不得写成本项目实验结论。P34 只是可读性增强，不代表证据等级提升。

## 下一步

后续如继续 P35，可优先做逐章人工个性化：减少跨章节重复句，把每章的引入案例、常见误读和练习说明进一步贴近具体章节对象。
