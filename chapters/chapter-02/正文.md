# 第 2 章 PyMOL、ChimeraX 与结构可视化

## 本章导读

第 1 章解决了环境、路径和记录问题。第 2 章把这些能力放到结构文件上：读者需要判断一个 PDB/mmCIF、AlphaFold 预测结构、对接构象或 Boltz2 输出结构，是否已经清楚到可以交给后续建模、对接、亲和力预测或蛋白设计。

结构可视化不是截图装饰。PyMOL 和 ChimeraX 的核心任务，是让链、配体、金属离子、水分子、缺失残基、口袋位置和结构来源变成可检查的记录。图像可以帮助读者看清结构对象，但不能替代来源说明、模型置信度、参数记录或实验验证。

本章正文依据全书大纲、第二章本章大纲、章节精读、方法笔记、AlphaFold 文献笔记、第二章原始学习素材和外部权威来源整理。原始 PDF、课件截图、Office 文件和压缩包只作为来源，不复制进正文。

本章采用一个固定贯穿案例：RCSB PDB `3HTB`，配体为 `JZ4`。PyMOL 是主线软件，负责选择、显示、测量、脚本化出图和最小复现；ChimeraX 是结构处理、表面/切片、核酸显示、光源材质和高级图形复核的补充工具。`1BNA` 只保留为 ChimeraX 核酸显示的补充练习，不作为本章主线案例。

固定案例的作用是让读者在同一个结构对象上反复练习“来源、链、配体、口袋和截图记录”。这不是把 `3HTB/JZ4` 当成药物设计项目结果，而是把它作为可追溯、可复查的教学材料。

| 字段 | 本章固定记录 | 正文用途 |
|:---|:---|:---|
| PDB ID | `3HTB` | 作为贯穿结构案例。 |
| 结构标题 | 2-propylphenol in complex with T4 lysozyme L99A/M102Q | 说明蛋白和配体对象。 |
| 方法与质量 | X-ray diffraction，1.81 A | 提供实验结构来源和分辨率记录。 |
| 蛋白对象 | T4 lysozyme L99A/M102Q，链 A | 作为 receptor chain 示例。 |
| 配体对象 | `JZ4`，2-propylphenol，`C9 H12 O`，formal charge 0 | 作为 ligand、口袋选择和互作复核示例。 |
| 来源页面 | <https://www.rcsb.org/structure/3HTB>；<https://www.rcsb.org/ligand/JZ4> | 用于下载、validation report 和 ligand interaction 复核。 |
| 检索日期 | 2026-06-06 | 记录正文使用时点。 |

本章完成后，读者应留下三类材料：结构来源表、可复现的口袋或界面截图、人工备注。第 3 章定义 docking box、第 5 章准备复合物输入、第 6 章提取设计约束时，都要回到这三类材料追问“这个结构假设从哪里来”。

## 学习目标

完成本章后，读者应能把“我看到了一个结构”改写成“我能说明这个结构能支持什么判断”。具体来说，应能做到：

- 区分实验结构、预测结构和人工处理后的工作结构。
- 在 PyMOL 中完成安装检查、默认设置、对象显示、选择、颜色、距离测量和截图导出。
- 用命令重建一个口袋视图，而不是只保存手动旋转后的图片。
- 说明分组、叠合、渲染和批量作图如何服务复现。
- 在 ChimeraX 中完成结构载入、蛋白/核酸显示、表面或切片视图和材质调整。
- 判断 PLIP、LigPlus、APBS、Caver 和 GetBox 输出最多支持什么结论。
- 把结构图写成包含来源、选择命令、视角、输出文件和边界说明的记录。

这些目标不要求读者一次记住所有命令。更现实的检查方式是：换一个结构文件后，读者仍能知道先看来源，再看链和配体，随后建立局部口袋视图，最后把截图、命令和边界写进记录。

## 核心概念

结构可视化的核心概念不是“按钮在哪里”，而是“这个视图约束了哪一种结构判断”。同一份结构文件可以用于整体折叠展示、口袋定位、界面残基整理或模型叠合，但每个用途需要不同记录字段。

| 概念 | 零基础解释 | 本章用法 |
|:---|:---|:---|
| 结构来源 | 结构文件从数据库、预测模型、对接输出或本地处理流程而来。 | 区分实验结构、AlphaFold 预测结构和工作结构。 |
| 链 ID | 大分子中不同分子链的编号。 | 决定后续选择、叠合、对接受体和界面解释。 |
| 配体与辅因子 | 小分子、离子、金属、核酸片段或非标准残基。 | 决定口袋定义、互作图和是否需要保留或删除。 |
| 选择集 | 在软件中命名的一组原子、残基或链。 | 把“我看到的区域”转成可复现命令。 |
| 显示模式 | cartoon、sticks、surface、mesh、dots 等视觉表示。 | 让读者分别看整体折叠、局部残基和表面形状。 |
| 叠合 | 把两个结构按指定区域对齐。 | 比较模型差异，但不能自动说明功能等价。 |
| 截图记录 | 保存视角、颜色、选择和图像文件。 | 作为结构复核证据的一部分，不能替代来源 metadata。 |
| 证据边界 | 当前材料最多能支持的结论范围。 | 防止把预测结构、口袋图或互作图写成实验结论。 |

读这张表时，应把每个概念对应到一个动作：结构来源要写入表格，链和配体要在软件中核对，选择集要有命令，截图要能重建，边界要写进图注或实验记录。

本章使用的证据判断遵循四层关系：材料显示什么，软件产生什么，允许怎样解释，还需要怎样验证。这个关系可以防止读者把颜色、距离或漂亮渲染误读为功能机制。

| 对象 | 材料显示什么 | 软件产生什么 | 允许怎样解释 | 还需要怎样验证 |
|:---|:---|:---|:---|:---|
| PDB/mmCIF 结构 | 坐标、链、配体、实验来源和部分质量信息。 | 结构视图、选择集、截图。 | 当前结构可作为某类输入候选。 | 检查缺失残基、生物学组装、突变和配体状态。 |
| AlphaFold 结构 | 序列到结构的预测模型和置信度线索。 | 按 pLDDT 或区域显示的模型视图。 | 可作为结构假设或补充线索。 | 核对低置信区域、配体缺失和复合物假设。 |
| 口袋截图 | 配体或残基附近的空间关系。 | 局部表面、sticks、距离标注。 | 可支持口袋定位和后续 box 设定线索。 | 需要来源、半径、残基列表和后续计算或实验。 |
| 互作图 | 自动识别的候选接触。 | 氢键、疏水、盐桥或距离列表。 | 可作为残基复核清单。 | 需要参数记录、结构质量和独立验证。 |

这张解释卡应在每次写图注前使用。若材料只显示一个预测模型，图注就不能写成实验结构；若软件只产生局部截图，正文就不能跳到结合能或活性解释。可视化工作真正交给后续章节的，是一个被限定过的结构假设。

这种限定不是削弱可视化的价值。相反，它让后续计算知道哪些对象已经人工看过，哪些只是占位输入，哪些还需要回到数据库、文献或实验记录补证据。读者越早写清边界，第 3 章和第 5 章越少出现“结构好像可以用但理由说不清”的问题。

## 方法流程

本章方法流程从结构载入开始，到可交接记录结束。不要先追求复杂图形；先让一个结构、一个配体和一个口袋能被别人复查。

| 步骤 | 输入 | 动作 | 输出 | QC/边界 |
|:---:|:---|:---|:---|:---|
| 1 | 结构文件 | 确认来源、下载日期、格式和置信度/分辨率。 | 结构来源表。 | 能追溯到 PDB ID、模型来源或本地流程。 |
| 2 | 链和配体 | 检查链 ID、配体、金属离子、水、缺失残基。 | 结构 QC 备注。 | 关键对象已命名，删除对象有理由。 |
| 3 | 局部区域 | 围绕配体、残基或界面建立选择集。 | 口袋或界面视图。 | 选择半径和残基列表可复现。 |
| 4 | 比较任务 | 按全局骨架、局部口袋或功能残基叠合。 | 叠合图和差异备注。 | 不用局部叠合推断整体功能。 |
| 5 | 输出任务 | 保存会话、命令、截图和人工判断。 | 可交接结构记录。 | 图注写明来源和边界。 |

执行时，优先完成一个小样例，再扩展到多个结构或批量截图。小样例通常会暴露最常见问题：配体名不一致、链 ID 错配、结构中没有共晶配体、预测模型低置信区域落在口袋附近，或截图没有保存选择命令。

### 2.1 PyMOL 安装、激活与基础设置

PyMOL 的安装目标很简单：软件能打开结构文件，并且默认设置不会让每次截图都依赖手动调整。Linux 课程素材给出的最小安装路径是通过系统包管理器安装并在终端输入 `pymol` 启动。

```bash
sudo apt update
sudo apt upgrade
sudo apt install pymol
pymol
```

安装成功只说明软件能运行，不说明结构已经处理正确。PyMOL 官方支持页（<http://pymol.com/support>）提供 Windows、macOS、Linux、Conda 安装、许可证和引用格式信息；正文引用 PyMOL 时，应按本机实际版本记录，而不是沿用示例版本号。第一次打开 PyMOL 后，建议先确认当前目录、输入文件路径、输出目录和 PyMOL 版本，随后再配置默认显示风格。

`.pymolrc` 是 PyMOL 的默认设置入口。Linux/macOS 常用 `~/.pymolrc`，Windows 可使用 `C:\Users\<你的用户名>\.pymolrc.py`。这里适合写入背景、光线、抗锯齿、颜色别名和常用宏，但不应把项目特异选择写死在全局配置里。

```pymol
bg_color white
set ray_trace_mode, 1
set ray_trace_fog, 0
set depth_cue, 0
set antialias, 2
set ambient, 0.7
set direct, 0.4
alias rw, remove solvent
```

| 设置对象 | 解决的问题 | 边界 |
|:---|:---|:---|
| 背景和光线 | 让截图干净、可读。 | 不改变结构证据等级。 |
| 抗锯齿和 ray tracing | 提高图片质量。 | 不能掩盖残基缺失或配体处理问题。 |
| 颜色别名 | 保持图件风格一致。 | 颜色含义需要图注说明。 |
| `alias rw` | 快速删除水分子。 | 删除前要确认水是否参与结构解释。 |

### 2.2 PyMOL 界面、选择、颜色和距离测量

PyMOL 界面可以分成两层：鼠标操作帮助观察结构，命令记录保证可复现。左键旋转、中键或 Shift+左键平移、滚轮缩放适合探索视角；正式记录时，应把关键对象转换成选择命令。

对象菜单中的 `A`、`S`、`H`、`L`、`C` 分别对应 Action、Show、Hide、Label、Color。初学者可以用菜单摸清对象层级，但交接给下一步计算时，仍应保留对应命令。

| 操作 | 典型命令 | 记录意义 |
|:---|:---|:---|
| 按链选择 | `select receptor_chain, chain A` | 明确受体链或比较对象。 |
| 按残基编号选择 | `select resi 10-20` | 明确功能残基或局部片段。 |
| 按残基名选择 | `select ligand_jz4, resn JZ4` | 定位配体、辅因子或水分子。 |
| 按距离选择 | `select pocket_5a, byres (receptor_chain within 5 of ligand_jz4)` | 建立配体附近残基集合。 |
| 排除氢或溶剂 | `select heavy, not name H`；`remove solvent` | 控制图面复杂度和分析对象。 |

```pymol
fetch 3htb, case_3htb, type=pdb, async=0
select receptor_chain, case_3htb and chain A
select ligand_jz4, case_3htb and resn JZ4
select pocket_5a, byres (receptor_chain within 5 of ligand_jz4)
show cartoon, receptor_chain
show sticks, ligand_jz4 or pocket_5a
color cyan, receptor_chain
color yellow, ligand_jz4
```

距离测量适合回答“哪些原子或残基靠得近”，不适合直接回答“是否形成稳定结合”。如果需要写互作判断，应同时记录结构来源、距离阈值、是否加氢、是否处理电荷，以及该距离是否来自单一静态结构。

### 2.3 PyMOL 分组、叠合、渲染与批量作图

当一个 session 中同时有实验结构、预测结构、配体、口袋残基和截图视图时，分组可以防止对象混乱。分组的价值不是整理界面，而是让读者知道哪些对象属于同一个判断任务。

```pymol
fetch 3htb, exp_3htb, type=pdb, async=0
load outputs/3htb_receptor_prepared.pdb, prepared_3htb
group source_models, exp_3htb prepared_3htb
align prepared_3htb and chain A and name CA, exp_3htb and chain A and name CA
show cartoon, source_models
show sticks, byres ((exp_3htb and chain A) within 5 of (exp_3htb and resn JZ4))
```

叠合前必须先说明比较对象。全局骨架叠合适合观察整体折叠差异，局部口袋叠合适合看活性位点附近构象，界面残基叠合适合比较 PPI 或复合物假设。不同叠合对象不能混用同一个结论。

| 图形任务 | 推荐做法 | 常见误读 |
|:---|:---|:---|
| 多模式显示 | 用 `show cartoon` 叠加 sticks 或 surface。 | `show as cartoon` 会替换旧显示方式。 |
| 结构叠合 | 先写清全局、局部或功能残基范围。 | 用局部相似推断整体功能一致。 |
| 高质量截图 | 固定视角、分辨率、背景和光线。 | 把美化图当作新证据。 |
| 批量出图 | 使用脚本统一输入、颜色、视角和输出。 | 批量图片没有来源表，无法复核。 |

渲染和批量作图应服务复现。一个可交接的截图至少要能回答：输入文件是什么，选择了哪些对象，隐藏了什么，颜色代表什么，输出路径在哪里，截图是否经过人工裁剪。

### 2.4 口袋、静电、疏水、动画和剖面图

口袋图是第 2 章和第 3 章之间最重要的交接物。本章用 `3HTB` 中的 `JZ4` 演示这个判断路径：先确认配体和受体链，再定义配体附近 5 A 内的邻近残基，最后生成局部视图。

```pymol
fetch 3htb, case_3htb, type=pdb, async=0
remove solvent
select receptor_chain, case_3htb and chain A
select ligand_jz4, case_3htb and resn JZ4
select pocket_5a, byres (receptor_chain within 5 of ligand_jz4)
hide everything
show cartoon, receptor_chain
show sticks, ligand_jz4 or pocket_5a
show surface, pocket_5a
```

静电、疏水、动画和剖面图的边界不同。它们可以帮助读者观察表面性质、通道、空间遮挡或构象变化，但不能单独说明结合强度、活性变化或转运动力学。

| 图形类型 | 需要的输入 | 能说明什么 | 不能说明什么 |
|:---|:---|:---|:---|
| 口袋图 | 配体、功能残基或预测口袋。 | 口袋位置、附近残基和展示视角。 | 配体一定结合或活性一定存在。 |
| 静电图 | 质子化、电荷、半径和表面参数。 | 当前参数下的表面电势线索。 | 真实结合能或实验亲和力。 |
| 疏水图 | 表面展示和疏水标注规则。 | 表面性质分布的观察线索。 | 疏水作用强度的定量结论。 |
| 动画 | 视角序列或轨迹帧。 | 观察构象变化或展示路径。 | 未经轨迹分析的动力学机制。 |
| 剖面图 | 切片位置、表面和内部空间。 | 空腔、通道或遮挡关系。 | 通道真实可通过或配体迁移速率。 |

静电图尤其容易被误读。若使用 APBS 或相关插件，应记录电荷和半径来源；若只用伪电荷或快速展示模式，正文应写成“展示用表面”，不能写成物化分析。

### 2.5 PyMOL 常用命令、脚本与插件

PyMOL 命令可以按任务分成五组：载入与保存、选择、显示与隐藏、颜色、删除与保留。初学阶段不要追求命令数量，先保证每条命令都能对应到结构记录字段。

| 任务 | 命令例子 | 写入记录时说明 |
|:---|:---|:---|
| 载入 | `fetch 3htb`；`load myprotein.cif` | 结构来源、文件格式和对象名。 |
| 保存 | `save myprotein.pdb` | 保存的是原始结构、处理后结构还是局部对象。 |
| 显示 | `show cartoon, all`；`show sticks, ligand_jz4` | 视觉层级和对象范围。 |
| 隐藏 | `hide everything, all`；`hide surface, all` | 隐藏的是显示方式，不等于删除数据。 |
| 颜色 | `color 0x8db9d7, case_3htb`；`spectrum b, blue_red, all` | 颜色含义和是否代表 B 因子或置信度。 |
| 删除 | `remove solvent`；`remove not (chain A)` | 删除理由和是否会影响后续计算。 |

脚本用于把一套操作从“手动做过”变成“别人能重跑”。课程素材提到整体图、局部放大图和批量绘图可在后续 VibeCoding 中扩展；本章只要求读者先写出单个结构的最小脚本。

插件应按“输入、输出、复核点”理解，而不是按软件名背诵。

| 插件/工具 | 典型用途 | 复核边界 |
|:---|:---|:---|
| PymolFold | 根据 UniProt ID 抓取 AlphaFold 结构并按置信度着色。 | 低置信区和缺失配体必须单独记录。 |
| Alignment | 辅助结构叠合与比较。 | 叠合范围决定解释范围。 |
| GetBox | 从小分子或口袋定位 docking box。 | box 坐标是输入假设，不是结合证据。 |
| APBS | 生成静电表面。 | 依赖质子化、电荷和半径处理。 |
| Caver3 | 分析蛋白通道或隧道。 | 起点、探针半径和聚类阈值会影响结果。 |
| Chatmol | 辅助生成图形或命令。 | 可复现性和速度需要人工判断。 |

例如，Caver3 的最小探针半径、壳层深度、壳层半径、聚类阈值和起始点优化都会影响通道结果。首次分析不应把自动找到的通道直接写成配体迁移机制，而应写成“在该参数下识别到候选通道，需要进一步复核”。

插件安装不在本章正文展开。单独练习页应记录每个插件的安装来源、版本、输入文件、输出文件、失败信息和人工复核；本章只说明这些插件在结构复核链条中的用途和证据边界。

### 2.6 ChimeraX 安装与基础操作

ChimeraX 是 PyMOL 的补充，而不是 PyMOL 的替代品。课程素材把它放在 PyMOL 之后，主要用于某些图形方式、表面、核酸显示、切片和更强的交互式查看。安装和引用信息应以 PyMOL 官方支持页、ChimeraX User Guide 和 ChimeraX Citing 页面为准，具体版本以本机安装记录为准。

ChimeraX 的软件论文可用于说明工具来源和功能范围。Pettersen et al. 2021（DOI `10.1002/pro.3943`）侧重结构可视化用户、教育和开发场景；Meng et al. 2023（DOI `10.1002/pro.4792`）侧重结构构建和分析工具。它们支持“ChimeraX 可用于这些可视化和分析任务”的说法，不支持对 `3HTB/JZ4` 口袋或任何项目结构作药物设计结论。

ChimeraX 的基本流程同样从结构来源开始。读者可以先完成一个最小任务：打开结构、隐藏无关对象、显示 cartoon 或 surface、保存截图，再把命令写进记录。

```chimerax
open 3htb
hide solvent
show cartoon /A
style :JZ4 stick
surface /A
color bychain
view :JZ4
save outputs/3htb_chimerax_view.png width 1800 height 1200 supersample 3
```

| PyMOL 更适合 | ChimeraX 更适合 |
|:---|:---|
| 快速选择、命令复现和批量截图。 | 交互式结构处理、表面、切片和材质。 |
| 简洁展示配体附近残基。 | 复杂复合物、核酸、体积或表面视图。 |
| `.pml` 脚本管理图片风格。 | 场景、光源和高级渲染调整。 |

真实项目中，建议同一结构至少用一个工具完成主图，用另一个工具做复核。两者若显示出不一致的链、配体或表面对象，应先回到结构文件和选择命令，而不是优先调整截图效果。

### 2.7 ChimeraX 蛋白、核酸、光源和材质设置

ChimeraX 的价值在于让复杂结构更容易检查。蛋白整体可以用 cartoon 和 surface；核酸可以用 `1BNA` 这类简单结构练习碱基、骨架和沟槽的显示；复合物可以用不同链色和透明表面检查界面遮挡。

| 展示任务 | 观察重点 | 记录字段 |
|:---|:---|:---|
| 蛋白 cartoon | 二级结构、结构域和缺失区域。 | 链 ID、残基范围、结构来源。 |
| 核酸视图 | 双螺旋、链方向、沟槽和配体位置。 | 核酸链、配体、离子和显示模式。 |
| 表面视图 | 口袋形状、界面遮挡、整体体积。 | 表面对象、透明度和选择范围。 |
| 光源材质 | 读者能否看清关键区域。 | 背景、光源、材质和截图文件。 |
| 切片/剖面 | 内部空腔、通道或结合口袋。 | 切片位置、视角和不能推出的结论。 |

光源和材质设置只改变读者看到的方式，不改变结构本身。图注应说明颜色、透明度和显示模式代表什么；如果颜色只服务可读性，不代表 pLDDT、B 因子、电荷或残基类别，也要明确写出。

### 2.8 PLIP、LigPlus 等互作作图工具

PLIP、LigPlus、VMD、Grace 等工具适合放在“辅助作图和结果整理”层。它们可以帮助读者把氢键、疏水接触、盐桥、距离或轨迹曲线整理成图，但不能自动替代结构来源审查。

| 工具 | 常见输出 | 教材中的稳健用法 |
|:---|:---|:---|
| PLIP | 蛋白-配体相互作用列表和二维/三维线索。 | 生成候选接触清单，再回到结构中复核。 |
| LigPlus | 二维互作图。 | 辅助图注和残基命名，避免直接声明机制。 |
| VMD | 轨迹与结构展示。 | 轨迹可视化入口，本章只作工具位置说明。 |
| Grace | 曲线和轨迹指标图。 | 用于展示数值曲线，不替代统计分析。 |

互作图最容易被误写成结论。稳健写法是“PLIP 在该结构中识别出若干候选接触，后续需要结合距离阈值、质子化状态、结构质量和实验/计算验证解释”，而不是“该配体通过这些相互作用产生药效”。

## 代码案例与软件操作

本节训练一个最小复现任务：用 PyMOL 从 `3HTB` 中导出 `JZ4` 附近 5 A 口袋截图，并留下能够交接给第 3 章的选择命令。真实项目可以保留这套记录格式，但必须替换为自己的结构来源、链 ID、配体名、选择半径和输出路径。

```pymol
fetch 3htb, case_3htb, type=pdb, async=0
hide everything, case_3htb
select receptor_chain, case_3htb and chain A
select ligand_jz4, case_3htb and resn JZ4
select active_site_5a, byres (receptor_chain within 5 of ligand_jz4)
show cartoon, receptor_chain
show sticks, ligand_jz4 or active_site_5a
color slate, receptor_chain
color yellow, ligand_jz4
png outputs/3htb_jz4_active_site.png, dpi=220
```

| 检查项 | 合格记录 |
|:---|:---|
| 输入 | 写明结构来源、文件路径、下载日期和是否经过处理。 |
| 对象 | 写明链 A、配体 `JZ4`、金属离子、水分子和删除对象。 |
| 选择 | 写明 `resn JZ4`、`within 5` 等选择规则。 |
| 输出 | 保存图片、会话或脚本，并记录 `outputs/3htb_jz4_active_site.png` 或实际输出路径。 |
| 边界 | 图注说明截图只能支持结构复核，不说明结合或活性。 |

这段代码生成的是口袋截图，不是 docking 结果。它支持读者说明“我如何选择配体附近 5 A 残基”，不能支持“该配体一定结合”或“该口袋具有实验活性”。

固定案例还应留下一个简短记录，便于后续章节接收。

| 字段 | 示例记录 |
|:---|:---|
| 结构来源 | RCSB PDB `3HTB`，检索日期 2026-06-06。 |
| 受体对象 | T4 lysozyme L99A/M102Q，链 A。 |
| 配体对象 | `JZ4`，2-propylphenol，`C9 H12 O`。 |
| 选择命令 | `select active_site_5a, byres (receptor_chain within 5 of ligand_jz4)` |
| 输出文件 | `outputs/3htb_jz4_active_site.png` |
| 解释边界 | 该截图支持口袋定位和后续 docking box 设定线索，不支持结合强度或活性结论。 |

## 实验/练习入口

本章练习的重点是把一次结构观察转化成可交接记录。练习完成后，另一位同学应能根据你的表格、命令和截图复现结构来源到口袋视图的判断链。

1. 先用 `3HTB/JZ4` 完成固定案例记录，写出结构来源、链 ID、配体、分辨率和检索日期。
2. 再选择一个自己的 PDB/mmCIF 或 AlphaFold 结构，写出结构来源、链 ID、配体、分辨率或置信度。
3. 在 PyMOL 中导出一张配体附近 5 A 口袋图，保存选择命令和输出路径。
4. 在 ChimeraX 中打开同一结构，用不同显示模式复核链、配体和表面视图。
5. 选择 PLIP、LigPlus、APBS、GetBox 或 Caver 中任一工具，写出输入、输出和不能推出的结论。

PyMOL 插件安装另设练习页。本章练习只要求读者说明插件在结构复核中的位置，不要求在正文中展开 PymolFold、Alignment、GetBox、APBS、Caver3 或 Chatmol 的安装过程。

| 练习产物 | 必填字段 |
|:---|:---|
| 结构来源表 | 结构 ID、来源、格式、下载日期、质量备注。 |
| PyMOL 脚本 | 输入路径、对象名、选择命令、截图路径。 |
| ChimeraX 备注 | 打开命令、显示模式、复核对象。 |
| 图注草稿 | 图中对象、颜色含义、选择范围、证据边界。 |
| 待确认项 | 低置信区域、缺失残基、配体状态、是否需要实验验证。 |

如果练习借用了课程示例或文献案例，应把它写成方法参照。只有当输入结构、命令、截图和人工判断都来自本项目记录时，才适合进入后续受体准备或对接流程。

## 使用边界与常见误读

本章最容易误读的对象是结构图、预测模型、叠合图、静电表面和自动互作图。它们可以支持结构复核和假设整理，但不能单独说明结合强度、功能机制或药效。

| 易误读对象 | 稳健表述 | 不应表述 |
|:---|:---|:---|
| 漂亮结构图 | 该图展示了当前结构文件中的链、配体和口袋视图。 | 该图足以说明结构可靠。 |
| AlphaFold 结构 | 预测结构可作为结构假设，需结合置信度和场景解释。 | 预测结构等同实验结构。 |
| 局部叠合 | 在指定区域观察到构象相似或差异。 | 两个蛋白功能相同或活性相同。 |
| 静电/疏水图 | 当前参数下提供表面性质线索。 | 表面颜色说明结合强弱。 |
| PLIP/LigPlus 图 | 生成候选接触清单，供人工复核。 | 自动互作图说明作用机制。 |
| GetBox 坐标 | 可作为 docking box 初始假设。 | box 坐标说明口袋真实可结合。 |

结构图的证据边界应停在“可视化复核支持的结构观察”。当结论涉及结合、活性、转运动力学、亲和力或临床意义时，需要引入后续计算、文献或实验依据。

写作时，推荐使用“可用于定义候选口袋”“提示该区域需要复核”“在该参数下识别到候选接触”等表达。避免把截图、dry-run、预测结构或插件输出写成项目实验结论。

如果读者不确定某句话是否越界，可以把它拆成两句检查。第一句只写软件中看到了什么，例如“配体附近 5 A 内包含若干疏水残基”；第二句再写下一步需要什么，例如“这些残基是否稳定参与结合，仍需结合对接、动力学、能量计算或实验信息判断”。这种写法能保留观察价值，也能避免把单张静态图放大成机制。

在报告和论文图注中，颜色、距离和标签都应有明确定义。若颜色代表链或展示风格，就不要让读者误以为它代表电荷、置信度或活性；若距离来自一个静态构象，就不要写成稳定氢键。结构可视化可以启动判断，但不应独自结束判断。

## 延伸阅读与下一步

完成本章后，读者应把结构复核材料交给第 3 章。第 3 章会继续处理结构建模、结合位点和体系准备；如果本章记录缺少结构来源、链 ID、配体状态或口袋依据，应先补齐，不要直接运行更复杂的软件。

| 交接到下一章的材料 | 下一章如何使用 |
|:---|:---|
| 结构来源表 | 判断采用实验结构、预测结构还是人工处理结构。 |
| 口袋截图和选择命令 | 定义结合位点、受体范围和 docking box。 |
| 链/配体/金属离子备注 | 决定受体准备时保留、删除或单独处理的对象。 |
| 叠合和模型比较备注 | 判断是否需要同源建模、结构替换或多结构候选。 |
| 证据边界说明 | 防止后续 docking score 或 predicted affinity 被过度解释。 |

进入第 3 章前，读者至少应能回答四个问题：结构从哪里来，使用哪条链和哪个配体，口袋依据是什么，哪些结论仍需验证。只要其中一项答不上来，就先回到本章补记录。

如果后续任务是分子对接，本章输出应进一步整理成受体准备清单：保留链、删除对象、共晶配体、金属离子、水分子处理、box 中心和 box 尺寸。如果后续任务是蛋白设计，本章输出应整理成 motif、hotspot、界面残基和需要固定的结构片段。

轨迹可视化、代表构象截图和构象叠合仍然使用同一套原则：先说明输入来自哪里，再说明显示了什么，最后说明哪些解释需要轨迹指标、能量分析或实验信息支持。这样，第 2 章学到的不是一个软件界面，而是一套结构证据交接方式。
